# 基环树

## 无向图的基环树

1. 图中有 `n` 个点、 `n` 条边，图是连通，恰好有一个环。
2. 一个环 + 环上挂树的结构。
3. 每棵分树建立信息，然后在环上整合信息，进而得到答案。
4. 分树建立信息，树相关的算法和数据结构可能用到。
5. 环上整合信息，先把环变成链，然后很多数据结构就可以维护环的信息了。

## 有向图的基环树

1. 内向树，每个节点以自己为起点的边只有一条，从外进入环。
2. 外向树，每个节点以自己为终点的边只有一条，从环去往外。

## 三种找环方式

1. dfs找环：扩展性最好，可以得到环上的点、环上的边、环的顺序、环的边权等信息。
2. 并查集找环：适用于无向图，边依次加入，如果一条边的两个端点已经连通，这条边就是形成环的边，并查集通常只能找到闭环边，不能直接得到环上所有点。
3. 拓扑排序找环：拓扑排序适合标记哪些点在环上，但环的顺序、环边、环边权等组织信息无法直接获得。
   - 无向基环树：度为 `1` 的点进队列，消除该点的影响，最后剩下的点就是环上的点。
   - 有向基环树：入度为 `0` 的点进队列，消除该点的影响，最后剩下的点就是环上的点。

## dfs找环代码实现

1. dfs过程中，每个节点分配 `dfn` 序号， `dfn[u]` 表示第一次来到节点 `u` 时，该点获得的序号。
2. dfs过程中，每个节点建立 `from` 信息， `from[u]` 表示当前dfs路径中，节点 `u` 的上一个节点。
3. 当来到节点 `u` ，枚举 `u` 的儿子 `v` ，如果发现 `v` 之前遍历过，又有 `dfn[u] < dfn[v]` ，说明发现了环。
4. 环为 `u -> .. -> v -> u` ，可以利用 `from` 信息，得到整个环。
5. 不仅想得到环上点的信息，还想获得环上依次出现边的信息，扩充 `from` 信息即可。
6. 找到环不要剪枝退出，而是遍历当前连通块的所有节点，都获得 `dfn序号` ，方便找到下一个连通块。

```java
void dfs(int u) {
    dfn[u] = ++cntd;
    for (int e = head[u]; e > 0; e = nxt[e]) {
        int v = to[e];
        if (dfn[v] == 0) {
            from[v] = u;
            dfs(v);
        } else if (dfn[u] < dfn[v]) {
            此时发现环了，u是起点，v是终点
            利用from，收集环上节点，或者环的其他信息
        }
    }
}
```
